# Технический долг

## Инструмент `PROJECT_FS_SUMMARY` (исключен из мультиагентного режима)

### Статус
- Инструмент временно отключен в мультиагентном режиме `AiAgentMain`.
- Все упоминания `PROJECT_FS_SUMMARY` удалены из мультиагентных контрактов (prompt/parser/models/tool gateway).

### Почему это долг
- Ранее инструмент возвращал краткий снимок файловой системы проекта (root/tree/snippets), но без формального контракта качества, ограничений и критериев применимости.
- Это приводило к смешиванию ответственности между оркестратором и инструментами, а также к нестабильному качеству контекста для шагов субагентов.

### Что нужно реализовать
1. Выделить отдельный проектный инструмент обзора ФС с четким контрактом входа/выхода:
- вход: путь проекта, лимиты глубины и объема;
- выход: структурированный JSON (root, tree, selected_files, snippets, truncation_flags).
2. Добавить безопасные ограничения:
- лимит на размер ответа;
- исключение бинарных/крупных файлов;
- фильтрация чувствительных файлов и маскирование секретов.
3. Ввести детерминированную стратегию отбора:
- правила ранжирования файлов для включения в snippets;
- предсказуемое поведение при больших репозиториях.
4. Добавить telemetry/trace:
- причина исключения файлов;
- статистика покрытия и среза данных.
5. Добавить минимальные тесты:
- пустой каталог;
- крупный каталог с ограничением;
- наличие приватных данных;
- детерминизм результата.

### Где реализовать
- Новый профильный модуль/файл в `aiagentmain` (без раздувания `AiAgentMainScreen.kt`):
  - отдельный executor + formatter + модель результата.
- Интеграция через `AiAgentMainToolGateway` как отдельный tool kind только после стабилизации контракта.
- Включение в мультиагентный контур только после обновления:
  - `AiAgentMainMultiAgentModels.kt`
  - `AiAgentMainMultiAgentPrompts.kt`
  - `AiAgentMainMultiAgentParser.kt`
  - `AiAgentMainMultiAgentOrchestrator.kt`
