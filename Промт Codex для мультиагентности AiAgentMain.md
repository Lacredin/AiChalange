2) Оркестратор и субагенты
- Реализуй оркестратор, который после запроса пользователя принимает решение:
  - `DIRECT_ANSWER`: ответить самостоятельно.
  - `DELEGATE`: делегировать одному или нескольким субагентам.
  - `NEED_CLARIFICATION`: запросить уточнение у пользователя.
  - `IMPOSSIBLE`: сообщить о невозможности выполнения.
- При `DELEGATE` оркестратор сначала строит план выполнения (шаги, исполнитель шага, ожидаемый результат).
- Выполняй шаги плана, получай ответы субагентов, агрегируй, валидируй.
- По итогам валидации оркестратор должен уметь:
  - отдать готовый ответ;
  - отправить на доработку (rework);
  - запросить уточнение у пользователя и после этого продолжить доработку;
  - завершить с невозможностью выполнения.
- Если выбран `DIRECT_ANSWER`, оркестратор сразу дает ответ без планирования через субагентов.

- Стартовый набор субагентов (можно расширяемый):

  - `implementer` — подготовка решения/черновика.
  - `validator` — проверка полноты, рисков, несоответствий.
