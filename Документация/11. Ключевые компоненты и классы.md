# Ключевые компоненты и классы
Этот файл даёт концентрированную карту главных компонентов и объясняет, как они связаны между собой.
Актуализировано: 01.04.2026.

## Корневой запуск и навигация
- `main` (`main.kt`) — создаёт desktop window и запускает `RootApp`.
- `RootApp` (`RootApp.kt`) — переключает экраны по `RootScreen`, сохраняет последний выбранный экран.
- `RootScreen` — единый enum маршрутов внутри приложения.

## Сетевой слой
- `NetworkClient` — общий `HttpClient` с таймаутами, JSON, SSE, WebSocket и логированием.
- `DeepSeekApi`, `OpenAiApi`, `GigaChatApi`, `ProxyOpenAiApi` — провайдерные chat-клиенты.
- `LocalLlmApi` — локальный провайдер для Ollama-совместимого endpoint.
- `StreamingChatParser` (`parseStreamingChatResponse`) — общая логика обработки stream-ответов.

Связи:
- Все экраны чата используют эти API-клиенты напрямую.
- Формат сообщения единый: `DeepSeekMessage`.

## MCP и инструменты
- `RemoteMcpService` — клиент MCP-серверов, list/call инструментов, поддержка WebSocket и auxiliary WebSocket.
- `McpToolInfo` — DTO для списка инструментов.

Связи:
- `AiAgentMCPScreen` использует сервис для инспекции инструментов.
- `AiAgentMainScreen` использует сервис в execution-шаге при наличии `tool_request`.

## Экранные домены
- `App` — минимальный чат.
- `AiAgentScreen` — расширенный чат с контекстными стратегиями и ветками.
- `AiWeek3Screen` — профильная версия `AiAgent` с `LongTermMemoryEntry`.
- `AiAgentRAGScreen` — retrieval-чат с сущностями `RetrievedChunk`, `RetrievalConfig`, `RagTaskState`.
- `EmbedingGenerationScreen` — генератор эмбеддингов и чанков.
- `AiStateAgentScreen` — state-machine агент с `ValidationFailureInfo` и инвариантами.
- `AiAgentMainScreen` — state-machine + MCP; ключевая сущность для инструментов: `ExecutionToolRequest`.

## Модельные классы агентных сценариев
- `PlanJson`, `PlanStepJson` — структура плана, используемая в planning/execution/validation цикле.
- `AgentState` — жизненный цикл выполнения.
- `AiAgentMessage` — универсальная запись сообщения в рамках ветки выполнения.
- `InvariantEntry` — правило-ограничение, проверяемое на плане и на execution-результате.

## Команды и PR-review в `AiAgentMain`
- `AgentSlashCommandParser`, `AgentSlashCommand`, `agentCommandSuggestions` — парсинг и подсказки slash-команд.
- `AgentPrReviewUseCase` — orchestration сценария `/review-pr`.
- `AgentMcpPrReviewGitGateway` — MCP-шлюз к `git.pr_review_context` и `scm.submit_pr_review`.
- `AgentAuthTokenUseCase` — обработка `/auth-token` через `auth.upsert_token`.
- `AgentStructuredPrReview`, `AgentStructuredReviewPoint` — контракт структурированного AI-ревью.
- `buildAgentPrLexicalRagPayload` (`AiAgentMainPrReviewRag.kt`) — lexical fallback по документации и файлам PR.
- `buildAiAgentMainRagPayload` (`AiAgentMainRagRetrieval.kt`) — semantic retrieval по embedding-базе.

## Data/DB фабрики
- `DatabaseFactory`, `AiWeek3DatabaseDriverFactory`, `AiStateAgentDatabaseDriverFactory`, `AiAgentMainDatabaseDriverFactory`, `AiAgentRagDatabaseDriverFactory`, `EmbedingGenerationDatabaseDriverFactory`.

Роль:
- Создание SQLite драйвера.
- Проверка совместимости схемы.
- Для части баз — подготовка служебных таблиц (`app_settings`) и fallback-пересоздание.

## Вспомогательные компоненты
- `EmbeddingGeneratorStub` — генерация embedding-векторов через локальный endpoint, хранение выбранной embedding-модели.
- `chunking` (`chunkFixed`, `chunkStructured`, `chunkSemantic`) — стратегии разбиения текста на чанки.

Вероятно, эти компоненты уже являются кандидатом на вынос в отдельный reusable слой, так как используются в RAG-контуре и подготовке индекса.
