# Сетевые интеграции и MCP
Этот файл описывает внешние интеграции проекта и то, как реализован сетевой слой.
Актуализировано: 01.04.2026.

## Поддерживаемые LLM-провайдеры
- `DeepSeekApi` (`https://api.deepseek.com/v1`)
- `OpenAiApi` (`https://api.openai.com/v1`)
- `GigaChatApi` (`https://gigachat.devices.sberbank.ru/api/v1`)
- `ProxyOpenAiApi` (`https://openai.api.proxyapi.ru/v1`)
- `LocalLlmApi` (локальный endpoint `http://localhost:11434/api/chat`)

Во всех экранах провайдер выбирается пользователем, а запрос приводится к общей структуре `DeepSeekChatRequest`.

## Потоковый режим
- Для сетевых провайдеров используется общий парсер `parseStreamingChatResponse`.
- Поддерживаются SSE-строки формата `data: ...` и финализация по `[DONE]`.
- Если провайдер вернул сразу полный объект ответа, парсер тоже обрабатывает этот случай.
- Для локального LLM используется отдельный line-by-line потоковый формат через `LocalLlmApi`.

## Базовый HTTP-клиент
`NetworkClient.httpClient`:
- `SSE` и `WebSockets` включены одновременно.
- Настроены таймауты до 300 секунд для долгих агентных сценариев.
- Логирование `BODY`-уровня включено, заголовок `Authorization` санитизируется.

## Интеграция MCP
`RemoteMcpService` поддерживает:
- соединение через Streamable HTTP transport;
- соединение через WebSocket transport;
- `listAvailableTools`;
- `callTool`;
- `callToolViaWebSocket`;
- `callToolWithAuxWebSocket`.

Особенности:
- Есть эвристика `looksLikeWebSocketTool` для определения инструментов, требующих WS.
- Есть нормализация URL между `http/https` и `ws/wss`.
- Есть логирование направлений событий (`CONNECT`, `SEND`, `RECV`, `ERROR`, `CLOSE`) для UI.

## Где используется MCP
- `AiAgentMCPScreen` — ручная проверка доступности инструментов.
- `AiAgentMainScreen` — автоматический вызов MCP-инструментов во время execution-шага, если модель возвращает `tool_request`.

## MCP-инструменты для Git/PR workflow (`AiAgentMain`)
- `git.pr_review_context` — получение контекста PR (метаданные, diff, измененные файлы, содержимое файлов).
- `scm.submit_pr_review` — публикация итогового ревью в pull request.
- `auth.upsert_token` — сохранение/обновление GitHub токена для `repo`, `owner` или `global` binding.

Особенности:
- При ошибках авторизации (`AUTH_TOKEN_MISSING` и смежные коды) `AiAgentMain` пытается автоматически выполнить `auth.upsert_token` и повторить вызов PR-инструмента.
- Сервер для конкретного инструмента выбирается из активных MCP-серверов по доступному списку tools.

## Локальные зависимости окружения
- Для `LocalLlmApi` и `EmbeddingGeneratorStub` ожидается локально поднятый сервис Ollama на `localhost:11434`.
- Для MCP тестовых сценариев используются удаленные и локальные endpoint (например, `127.0.0.1:8080`).
