# Экран EmbedingGeneration
Этот файл описывает экран `EmbedingGeneration`, который подготавливает корпус для RAG.

## Назначение экрана
`EmbedingGeneration` — экран предварительной обработки источников: загрузка файлов, чанкинг, генерация эмбеддингов, запись результатов в локальную БД.

## Ключевые функции
- Выбор файлов через диалог.
- Поддержка форматов: текстовые + `pdf`.
- Выбор стратегий чанкинга:
  - фиксированная;
  - структурная;
  - семантическая.
- Пакетная обработка нескольких файлов.
- Отображение прогресса по каждому файлу.
- Хранение результата в `embedding_chunks`.
- Просмотр уже сохраненных чанков и фильтрация.
- Выбор и сохранение embedding-модели.

## Зависимости
- База: `createEmbedingGenerationDatabase(EmbedingGenerationDatabaseDriverFactory())`.
- Генератор эмбеддингов: `EmbeddingGeneratorStub`.
- Обработка PDF: `PDFBox` (`Loader`, `PDFTextStripper`).
- Вспомогательные стратегии чанкинга (как локальные функции экрана и общий пакет `chunking`).

## Какие данные использует
- Источник файла (`source`), имя, состояние обработки, прогресс.
- Текстовые секции документа.
- Чанки + embedding JSON.
- Настройки выбранных стратегий и модели (через `Preferences`).

## Логика обработки
- Текст читается из файла.
- Выделяются секции и общий текст для чанкинга.
- Для каждой активной стратегии строятся чанки.
- Для каждого чанка запрашивается embedding.
- Записи сохраняются в таблицу `embedding_chunks` с метаданными (`source`, `title`, `section`, `strategy`, `chunk_id`).
- Для источника предыдущие чанки удаляются перед новой обработкой.

## Возможности
- Полный локальный цикл подготовки данных для RAG без внешнего сервера индексации.
- Несколько стратегий деления текста для сравнения качества retrieval.
- Подготовленные данные сразу доступны экрану `AiAgentRAG`.

## Недостатки и ограничения
- Экран сочетает тяжелую обработку, логику БД и UI.
- Есть дублирование chunking-логики с пакетом `chunking`.
- Качество зависит от корректности выделения секций и точности PDF-извлечения.
- При крупных корпусах может расти время и стоимость локальной обработки.
