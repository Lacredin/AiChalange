# Данные и базы
Этот файл описывает модель хранения данных и роль каждой базы в проекте.

Проект использует SQLDelight и несколько отдельных SQLite-файлов по функциональным областям.

## Общие принципы
- Почти каждый крупный экран имеет отдельную БД.
- Для чатов обычно хранится:
  - `chats`;
  - `chat_messages`;
  - `chat_branch_messages`;
  - и дополнительные таблицы в зависимости от экрана.
- Драйверы БД создаются через `DatabaseFactory*.jvm.kt`.

## Состав баз и их назначение
- `app.db` (`DatabaseFactory.jvm.kt`):
  - базовые таблицы `chats`, `chat_messages`, `chat_feature_state`, `chat_branch_messages`;
  - используется экраном `AiAgent`.
- `ai_week3.db`:
  - профили (`chat_profiles`) и память (`memory_entries`);
  - используется `AiWeek3`.
- `ai_state_agent.db`:
  - профильная память + `invariant_entries`;
  - используется `AiStateAgent`.
- `ai_agent_main.db`:
  - профильная память + `invariant_entries` + `app_settings`;
  - используется `AiAgentMain`.
- `ai_agent_rag.db`:
  - чатовая история для RAG-экрана, `app_settings`;
  - используется `AiAgentRAG`.
- `embeding_generation.db`:
  - таблица `embedding_chunks` (`source`, `title`, `section`, `chunk_id`, `strategy`, `chunk_text`, `embedding_json`);
  - используется `EmbedingGeneration` и `AiAgentRAG`.

## Важные особенности схем
- В `AiWeek3`, `AiStateAgent`, `AiAgentMain`, `AiAgentRAG` есть `selected_profile_id` у чата.
- В `AiStateAgent` и `AiAgentMain` инварианты вынесены в отдельную таблицу и участвуют в валидации плана/выполнения.
- В `AiAgentMain` и `AiAgentRAG` добавлена таблица `app_settings` для персистентных UI/фича-флагов.

## Стратегия совместимости схем
По коду можно сделать вывод, что миграции не применяются. Вместо этого:
- проверяется наличие ожидаемых таблиц/колонок через `PRAGMA table_info`;
- при несовместимости база удаляется и создается заново.

Следствие:
- обновления схемы потенциально приводят к потере локальной истории.
- это технически просто, но рискованно для сценариев долгого использования.

## Потоки записи
- Пользовательское сообщение и ответ ассистента пишутся отдельными строками в `chat_messages`.
- Для state/branch логики параллельно пишутся записи в `chat_branch_messages`.
- Профильные настройки и фича-флаги обновляются через `upsert`-операции.
- Эмбеддинги пишутся пакетно при обработке файла.
