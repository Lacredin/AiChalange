# Экран AiAgentRAG
Этот файл описывает экран `AiAgentRAG`: retrieval-ориентированный чат на базе локально сохраненных эмбеддингов.

## Назначение экрана
`AiAgentRAG` предназначен для сценариев, где ответ должен формироваться на основе локального корпуса данных, а не только внутренних знаний модели.

## Ключевые функции
- Управление чатами RAG.
- Включение/выключение RAG-режима.
- Настройка retrieval-параметров:
  - `topKBefore`;
  - `topKAfter`;
  - `threshold`;
  - `useFilter`;
  - `useRewrite`.
- Формирование task-state из пользовательских сообщений:
  - цель;
  - уточнения;
  - ограничения;
  - термины.
- Retrieval по косинусной близости embedding-векторов.
- Форматирование ответа с обязательным блоком источников и цитат.
- Сравнительный режим двух конфигураций ответа.

## Зависимости
- База RAG-чатов: `createAiAgentRagDatabase(AiAgentRagDatabaseDriverFactory())`.
- База эмбеддингов: `createEmbedingGenerationDatabase(EmbedingGenerationDatabaseDriverFactory())`.
- Генерация embedding для запроса: `EmbeddingGeneratorStub`.
- Prompt-утилиты: `AiAgentRagPrompts`.
- Сетевые LLM-клиенты.

## Используемые данные
- Из `ai_agent_rag.db`:
  - чаты;
  - сообщения;
  - app settings (task-state на чат и прочее).
- Из `embeding_generation.db`:
  - `embedding_chunks` с текстом чанка и вектором.
- Временные:
  - активная retrieval-конфигурация;
  - состояние сравнения;
  - разворот источников для выбранного сообщения.

## Логика retrieval
- Запрос пользователя при необходимости переписывается (`useRewrite`).
- Генерируется embedding запроса.
- Все чанки ранжируются по cosine similarity.
- Применяются этапы `topKBefore` и опциональный фильтр по порогу.
- Формируется финальный набор `topKAfter`.
- Модель получает контекст + историю + task-state.
- Ответ постобрабатывается для принудительного соблюдения формата источников.

## Возможности
- Прозрачный и управляемый retrieval-пайплайн.
- Объяснимость ответа через источники и цитаты.
- Возможность сравнивать конфигурации retrieval в одном интерфейсе.

## Недостатки и ограничения
- Качество retrieval зависит от качества локальных эмбеддингов и стратегий чанкинга.
- Текущий формат цитирования контролируется в том числе эвристиками, а не только строгой структурой.
- При пустом/слабом корпусе данных RAG не дает устойчивой пользы.
- Экран содержит одновременно UI, retrieval-алгоритм и orchestration, что усложняет развитие.
