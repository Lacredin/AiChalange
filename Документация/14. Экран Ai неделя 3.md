# Экран Ai неделя 3
Этот файл посвящён экрану `AiWeek3`: расширенной версии `AiAgent` с профилями и долговременной памятью.

## Назначение экрана
`AiWeek3` развивается из `AiAgent` и добавляет персонализацию поведения чат-агента через профили и долгосрочные записи памяти.

## Ключевые функции
- Все функции `AiAgent` (контекстные стратегии, стрим, ветвление, история).
- Профили (`chat_profiles`) с независимыми настройками feature-флагов.
- Привязка конкретного чата к выбранному профилю (`selected_profile_id`).
- CRUD-операции по долгосрочной памяти (`memory_entries`) для профиля.
- Переключение профилей на активном чате без потери истории сообщений.

## Зависимости
- База: `createAiWeek3Database(AiWeek3DatabaseDriverFactory())`.
- SQLDelight-схема `aiweek3db` с таблицами:
  - `chats`;
  - `chat_messages`;
  - `chat_profiles`;
  - `chat_branch_messages`;
  - `memory_entries`.
- Сетевые LLM-клиенты и общие network-модели.

## Какие данные использует экран
- Данные чата и веток.
- Профили и их feature-состояние.
- Долговременная память профиля: пары `ключ: значение`.
- Текущее состояние UI-форм редактирования профиля и памяти.

## Внутренняя логика
- При открытии чата экран проверяет и восстанавливает привязанный профиль.
- Если профиль не задан, используется fallback-профиль.
- Параметры стратегий контекста загружаются из профиля, а не напрямую из чата.
- Long-term память может подмешиваться в системный контекст ответа.
- Изменения профиля синхронно отражаются в чатах, связанных с этим профилем.

## Возможности
- Режим «персон» для одного и того же движка модели.
- Отделение параметров поведения агента от конкретной истории сообщений.
- Накопление устойчивой памяти между чатами одной профильной конфигурации.

## Недостатки и ограничения
- Логика профилей и памяти интегрирована прямо в экран, что усложняет расширение.
- Нет отдельного контроля версий/качества данных памяти.
- В памяти нет типизации значений, только текстовые пары.
- Всё ещё сохраняется высокий объем UI+domain кода в одном файле.
