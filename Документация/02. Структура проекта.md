# Структура проекта
Этот файл фиксирует структуру каталогов и объясняет назначение каждого важного раздела.

Корень проекта:
- `build.gradle.kts`, `settings.gradle.kts`, `gradle.properties`, `gradle/libs.versions.toml` — общая конфигурация сборки.
- `secrets.properties.example` — шаблон секретов для API-ключей.
- `composeApp` — основной модуль приложения.

Структура `composeApp/src`:
- `commonMain/kotlin/com/example/aiadventchalengetestllmapi/network`
  - API-клиенты провайдеров и единый HTTP-клиент.
- `commonMain/kotlin/.../mcp`
  - `RemoteMcpService` для list/call инструментов MCP, включая WebSocket-режимы.
- `commonMain/kotlin/.../(db и *_db)`
  - `expect`-фабрики драйверов и функции создания SQLDelight-баз.
- `commonMain/sqldelight*`
  - SQL-схемы и SQLDelight-запросы для разных функциональных БД.
- `jvmMain/kotlin/com/example/aiadventchalengetestllmapi`
  - Основные экраны, `RootApp`, `main.kt`.
- `jvmMain/kotlin/.../aiagentmain`, `.../aiagentrag`, `.../chunking`, `.../embedding`
  - Специализированные экраны и утилиты.
- `jvmMain/kotlin/.../*db/DatabaseFactory.jvm.kt`
  - `actual`-реализация драйверов SQLite и стратегия проверки совместимости схем.
- `jvmMain/composeResources`
  - ресурсы desktop-приложения.

## Модульная картина
Физически модуль один (`composeApp`), но логически проект разделен на домены:
- `App` и `AiAgent` — базовые сценарии чата.
- `AiWeek3` — расширение чата профилями и памятью.
- `AiStateAgent` и `AiAgentMain` — state-machine агенты.
- `AiAgentRAG` + `EmbedingGeneration` — RAG-контур (подготовка индекса и retrieval).
- `AiAgentMCP` — инспекция MCP-инструментов.

## Точки входа
- `main.kt` — старт desktop-окна.
- `RootApp.kt` — корневой роутер по `RootScreen`.
