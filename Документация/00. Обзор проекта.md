# Обзор проекта
Этот файл описывает, что представляет собой проект, какую задачу решает и какие ключевые возможности уже реализованы.
Актуализировано: 01.04.2026.

Проект `AiAdventChalengeTestLlmApi` — это desktop-приложение на Compose Multiplatform (целевая платформа в текущем репозитории: JVM), ориентированное на эксперименты с AI-ассистентами, LLM API, RAG-подходом, извлечением эмбеддингов и сценариями «агентного» выполнения задач с валидацией шагов.

По структуре и логике приложение предназначено в первую очередь для разработчика или исследователя, который:
- сравнивает разные LLM-провайдеры на одних и тех же сценариях;
- тестирует стратегию контекста (суммаризация, sliding window, sticky facts, ветвление истории);
- строит retrieval-процесс поверх локального хранилища эмбеддингов;
- проверяет более сложные stateful-сценарии, где модель сначала планирует, затем выполняет шаги, потом проходит формальную и смысловую проверку;
- интегрирует MCP-инструменты в цикл выполнения шагов.

Основные функциональные области:
- Базовый multi-provider чат (`App`).
- Расширенный AI-чат с контекстными стратегиями и ветвлением (`AiAgent`).
- Версия с профилями и долгосрочной памятью (`AiWeek3`).
- RAG-экран с retrieval по эмбеддингам, фильтрацией и query rewrite (`AiAgentRAG`).
- Экран подготовки эмбеддингов и чанков из файлов (`EmbedingGeneration`).
- Экран просмотра MCP-инструментов (`AiAgentMCP`).
- State-agent сценарий с этапами планирования, исполнения, проверки и восстановлением после ошибок (`AiStateAgent`).
- Расширенный агент с MCP-вызовами в execution-цикле (`AiAgentMain`), агентными slash-командами и сценариями PR-review.
- Гибридный RAG для agent-режима: semantic retrieval по embedding-базе + lexical fallback по документации и файлам PR.

Что проект не делает «из коробки»:
- Нет полноценного production backend-слоя; всё orchestrated в клиентском desktop-приложении.
- Нет отдельного DI-контейнера (внедрение зависимостей в основном через `remember` и прямое создание объектов).
- Нет явного тестового покрытия ключевой логики в репозитории.
- Нет миграций схем БД в классическом виде; используется стратегия проверки совместимости схемы и пересоздания базы.
