# Модули и экраны
Этот файл описывает ключевые экраны и объясняет их роль в общей системе.

## `AppScreen` (`App`)
Назначение: минимальный универсальный чат для проверки API-провайдеров и параметров генерации.

Что делает:
- Позволяет выбрать API, модель и параметры (`temperature`, `top_p`, `max_tokens`, `presence_penalty`, `frequency_penalty`, `stop`).
- Поддерживает потоковый ответ и отображает время ответа.
- Не использует БД, работает как сессионный интерфейс.

## `AiAgentScreen`
Назначение: stateful-чат с локальной историей, контекстными стратегиями и ветвлением.

Ключевые возможности:
- Сохранение чатов и сообщений в базе.
- Переключение между стратегиями контекста:
  - суммаризация истории;
  - sliding window;
  - sticky facts (выделение устойчивых фактов в системный блок);
  - ветвление диалога.
- Ведение «raw» и «real» веток потока ответа.

## `AiWeek3Screen`
Назначение: развитие `AiAgent` с профилями поведения и долгосрочной памятью.

Дополнительно к `AiAgent`:
- Таблица профилей с feature-настройками.
- Привязка чата к профилю.
- `memory_entries` для long-term памяти по профилю.

## `AiAgentRAGScreen`
Назначение: чат с retrieval поверх локального хранилища эмбеддингов.

Ключевые функции:
- Настраиваемый retrieval (`topKBefore`, `topKAfter`, `threshold`).
- Опциональный query rewrite.
- Опциональная фильтрация по порогу сходства.
- Форматирование ответа с ссылками на источники и цитатами.
- Сравнительный режим (основная конфигурация против варианта без части фильтров).

## `EmbedingGenerationScreen`
Назначение: подготовка retrieval-базы из файлов.

Ключевые функции:
- Выбор файлов (`txt`, `md`, `csv`, `json`, `xml`, `log`, `pdf`).
- Извлечение текста (включая PDF через PDFBox).
- Чанкинг по стратегиям `Fixed`, `Structured`, `Semantic`.
- Генерация эмбеддингов через `EmbeddingGeneratorStub` (локальный Ollama).
- Сохранение чанков и embedding-векторов в `embedding_chunks`.

## `AiStateAgentScreen`
Назначение: агентный цикл «план → выполнение → проверка → восстановление».

Особенности:
- Ветви состояния: основная, планирование, выполнение, проверка.
- Формальная и смысловая валидация.
- Хранение и проверка инвариантов.
- Автоматическое восстановление после ошибок валидации.

## `AiAgentMainScreen`
Назначение: расширенный state-agent с поддержкой MCP-инструментов в execution-фазе.

Дополнительно к `AiStateAgent`:
- MCP-конфигурация серверов и инструментов.
- Парсинг `tool_request` из execution-ответа.
- Вызов MCP инструментов (HTTP/WebSocket/aux WebSocket).
- Логирование MCP-взаимодействий в чат и отдельные ветки.
- Хранение app-level флагов (например, включение инвариантов) в `app_settings`.

## `AiAgentMCPScreen`
Назначение: операционный экран для проверки доступности MCP-серверов и списка инструментов.

Функционал:
- Выбор сервера.
- Запрос `tools/list`.
- Визуализация доступных инструментов и их описаний.
